## 1. Contexto y Problemática

### Situación actual

Para crear un Glue Job con la gema, el caller debe:

1. Subir manualmente el script PySpark a S3:
   ```bash
   aws s3 cp mi_script.py s3://bucket/scripts/
   ```
2. Crear el job con la gema:
   ```ruby
   DataDrain::GlueRunner.create_job(
     "mi-job",
     script_location: "s3://bucket/scripts/mi_script.py",
     ...
   )
   ```

### Problema

Cada servicio que usa la gema debe repetir el paso de upload. No hay forma de definir scripts locales en el repo del servicio y que la gema se encargue del upload.

### Solución propuesta

Agregar un método `upload_script` y modificar `create_job`/`ensure_job` para aceitar `script_path` local. La gema sube el archivo a S3 automáticamente.

---

## 2. Decisión de Diseño

### Opción A: `script_path` en `create_job`/`ensure_job` (RECOMENDADA)

- Si se provee `script_location` → usa el path S3 directo (comportamiento actual)
- Si se provee `script_path` (local) → sube a S3 primero, luego usa ese path para crear el job

```ruby
# Uso actual (script ya en S3)
DataDrain::GlueRunner.create_job(
  "my-job",
  script_location: "s3://bucket/scripts/export.py",  # ya existe en S3
  role_arn: "arn:aws:iam::123:role/GlueRole"
)

# Uso nuevo (script local)
DataDrain::GlueRunner.create_job(
  "my-job",
  script_path: "scripts/glue/export.py",      # archivo local
  bucket: "my-bucket",
  script_folder: "scripts/",                  # destino en S3
  role_arn: "arn:aws:iam::123:role/GlueRole"
)
# → Sube scripts/glue/export.py a s3://my-bucket/scripts/export.py
# → Crea el job con script_location apuntando a s3://my-bucket/scripts/export.py
```

### Alternativas descartadas

- **Opción B: Método separado `upload_script`** — Requiere 2 pasos en el caller, más código duplicado.
- **Opción C: Bucket global en configuración** — Acopla el bucket al caller cuando podría variar por job.

## 3. API Propuesta

### Nuevo método: `GlueRunner.upload_script`

```ruby
DataDrain::GlueRunner.upload_script(
  local_path:,      # Path local al archivo (String)
  bucket:,          # Bucket S3 destino (String)
  script_folder:    # Folder S3 destino (String, default: "scripts/")
)
# => "s3://bucket/folder/filename.py"
```

**Comportamiento:**
- Valida que el archivo local existe
- Sube a S3 usando `Aws::S3::Client#put_object`
- Retorna el path S3 completo
- Idempotente: si el archivo ya existe con el mismo contenido, S3 lo sobrescribe (OK)

**Errores:**
- `DataDrain::ConfigurationError` si `local_path` no existe
- Errores AWS se propagan sin atrapar

### Modificación: `GlueRunner.create_job`

```ruby
def self.create_job(
  job_name,
  role_arn:,
  script_location: nil,    # S3 path directo (existing)
  script_path: nil,         # Path local (nuevo)
  bucket: nil,              # Bucket S3 para upload (requerido si script_path)
  script_folder: "scripts", # Folder S3 para upload
  ...
)
```

**Lógica:**
1. Si `script_path` && !`script_location`:
   - Requiere `bucket`
   - Llama `upload_script(local_path: script_path, bucket: bucket, script_folder: script_folder)`
   - Asigna resultado a `script_location`
2. Si `script_location` → usa directo (comportamiento actual)
3. Si `script_path` && `script_location` → error (ambigüedad)
4. Si `script_path` && !`bucket` → error

### Modificación: `GlueRunner.ensure_job`

Misma lógica que `create_job` — pasar `script_path`/`bucket`/`script_folder` directamente a `create_job`.

### 4.2 Archivo: `lib/data_drain/glue_runner.rb`

```ruby
# Near line 42 (after client getter)

# Upload a local script file to S3 and return the S3 path.
#
# @param local_path [String] Local path to the script file.
# @param bucket [String] S3 bucket name.
# @param script_folder [String] S3 folder path (default: "scripts").
# @return [String] The S3 path (e.g., "s3://bucket/scripts/my_script.py").
# @raise [DataDrain::ConfigurationError] if local_path does not exist.
# @raise [Aws::S3::Errors::ServiceError] if upload fails.
def self.upload_script(local_path:, bucket:, script_folder: "scripts")
  raise DataDrain::ConfigurationError,
        "Script local '#{local_path}' no existe" unless File.exist?(local_path)

  s3_client = Aws::S3::Client.new(region: DataDrain.configuration.aws_region)
  filename = File.basename(local_path)
  s3_key = File.join(script_folder, filename).gsub("\\", "/")

  File.open(local_path, "rb") do |file|
    s3_client.put_object(
      bucket: bucket,
      key: s3_key,
      body: file,
      content_type: "text/x-python"
    )
  end

  "s3://#{bucket}/#{s3_key}"
end

# Modificar create_job:
# Agregar params: script_location:, script_path:, bucket:, script_folder:
# Al inicio del método:
if script_path && !script_location
  raise DataDrain::ConfigurationError, "script_path requiere bucket" unless bucket
  script_location = upload_script(local_path: script_path, bucket: bucket, script_folder: script_folder)
elsif script_path && script_location
  raise DataDrain::ConfigurationError, "provee script_location o script_path, no ambos"
elsif !script_location && !script_path
  raise ArgumentError, "script_location o script_path es requerido"
end
```

    temp_script.write("# python script content")
    temp_script.close
    allow(Aws::S3::Client).to receive(:new).and_return(s3_client)
  end

  after { temp_script.unlink }

  it "sube archivo a S3 y retorna path correcto" do
    expect(s3_client).to receive(:put_object).with(
      bucket: "my-bucket",
      key: "scripts/export.py",
      body: an_instance_of(File),
      content_type: "text/x-python"
    )

    result = described_class.upload_script(
      local_path: temp_script.path,
      bucket: "my-bucket",
      script_folder: "scripts"

    expect(result).to eq "s3://my-bucket/scripts/export.py"

## 6. Risks y Edge Cases

| Risk | Mitigación |
|------|------------|
| Upload grande consume tiempo | El upload es rápido para scripts PySpark (típicamente <1MB). Si es problema, async o pre-upload. |
| Archivo local no existe | Validación con `File.exist?` + `ConfigurationError` descriptivo. |
| Bucket sin permisos de escritura | Error AWS propagado — caller debe tener permisos correctos. |
| Collision de nombres | `script_folder` + `basename(local_path)` — si hay 2 scripts con mismo nombre en folders distintos, el segundo sobrescribe. Documentar. |
| Encoding del archivo | Upload en binario (`"rb"`) preserva encoding. |
| Content-Type | Usar `text/x-python` explícito — S3 inferirá si no se provee. |
